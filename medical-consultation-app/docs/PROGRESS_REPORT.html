Tóm tắt kiến trúc (hiện tại)

  • /api/agent-chat là gateway Agent mode: UI bật Agent toggle ở /tu-van, gửi agent_id để chọn persona/profile.
  • Routing hybrid đọc SSOT data/runtime-mode.json để ưu tiên GPU/CPU; có fallback và ghi runtime-events.jsonl, runtime-metrics.jsonl.
  • Provider Agent: Gemini function calling, OpenAI-compatible JSON-agent (GPU/CPU), và FOZA (OpenAI-compatible tool_calls → actions + fallback an toàn).
  • Tool server-side theo pattern “mcp-lite”: agent gọi tool (web/youtube) → /api/mcp/call thực thi → trả kết quả cho agent tổng hợp.
  • Allowlist điều hướng chuẩn hoá bởi ALLOWED_PATH_PREFIXES; frontend chỉ thực thi action hợp lệ.

Trạng thái hiện tại

  • UI có Agent mode + chọn Agent Profile (tổng quát/triage/thuốc/kế hoạch/trị liệu).
  • conversationId chuẩn hoá UUID; chặn legacy để tránh lỗi DB.
  • Gemini STT/TTS server-side cho demo Live mode.
  • Provider chat thường: /api/llm-chat hỗ trợ server|gemini|foza.

Next steps (đề xuất demo/kiểm thử)

  • Demo triage: chọn profile “Triage + Red flags” → mô tả triệu chứng nguy hiểm → agent ưu tiên hướng dẫn an toàn + gợi ý mở bac-si.
  • Demo trị liệu: chọn “Tâm lý trị liệu” → yêu cầu bài tập giảm lo âu → agent gợi ý mở tri-lieu / sang-loc.
  • Demo tool calling: “tìm nguồn về …” (web.search) hoặc “gợi ý nhạc thư giãn” (youtube.recommend_music).
  • Nếu FOZA tool_calls không ổn định: chuyển sang chuẩn JSON-in-content {response, actions} cho Agent mode.
